#  数据容器

数据容器是一种可以容纳多份数据的数据类型,容纳的每一份数据称之为1个元素,每一个元素都可以是任意类型的数据

容器的类型
- 列表(list)
- 字符串(str)
- 元组(tuple)
- 集合(set)
- 字典(dict)

他们之间的区别体现在是否支持重复元素,是否可以修改,是否有序等

---

# 列表

列表属于数据容器的其中一种,一次性可以存储多个元素

定义:`列表名称 = [元素1,元素2,...]`

特点:
	- **可以存储不同类型的数据**
	- 元素**有序,可以重复,元素可以修改**

列表中的元素存在索引,从0开始,我们可以通过`s[0]`,`s[1]`这样的方式来操作列表中的元素

python还支持反向索引,当索引为负数时,表示这是一个反向索引,从后向前,`s[-1]`为列表中最后一个元素,-2,-3,以此类推

列表属于***序列***类型的一种,在此引出序列类型的定义:

容器中的元素按特定顺序排列的,可通过索引访问的容器类型称之为序列.

字符串和元组也都是序列类型

接下来讲解序列类型都支持的切片操作

---

## 切片

切片是指对操作的数据截取其中一部分的操作

语法:`序列数据[开始索引:结束索引:步长]
- 不包含结束索引对应的元素
- 开始索引未指定默认为0,结束索引未指定默认为序列长度(即对整个序列进行操作),步长未指定默认为1
- 索引采用正向反向均可

切片结果类型不变,例如对一个列表进行切片操作,得到的容器仍然为列表类型

---
## 列表的常见方法


| 方法          | 作用                             | 样例                             |
| ----------- | ------------------------------ | ------------------------------ |
| `append()`  | 在列表尾部添加元素                      | `s.append(30)`                 |
| `insert()`  | 在指定索引之前,插入该元素                  | `insert(0,90)# 在第一个元素之前添加元素90` |
| `remove()`  | 移除列表中第一个匹配的值                   | `s.remove(75)`                 |
| `pop()`     | 删除列表中制定索引位置的元素,如果未指定索引,默认删最后一个 | `s.pop(2)`<br>`s.pop()`        |
| `sort()`    | 对列表进行排序                        | `s.sort()`                     |
| `reverse()` | 反转列表元素                         | `s.reverse()`                  |
| `sum()`     | 对列表进行求和                        | `sum(list)`                    |
| `len()`     | 返回列表长度                         | `len(list)`                    |

注意:
-  对于`sort()`方法,只有列表元素的数据类型一致,才可以进行排序,默认从小到大
- `insert()`方法的两个参数,第一个参数表示索引,第二个参数表示要在索引前面插入的元素
- `pop()`方法会把删除的元素进行返回,例如`num = pop(2)`

#例题

合并两个列表中的元素,并对合并的结果进行去重处理(去除列表中的重复元素)

```python
#定义列表
num_list1 = [19,23,54,64,875,20,109,232,123,54]
num_list2 = [55,80,72,35,60,123,54,29,91]
```

1. 解法1:黑马程序员解法

```python
num_list1 = [19,23,54,64,875,20,109,232,123,54]
num_list2 = [55,80,72,35,60,123,54,29,91]

for num in num_list2:
	num_list1.append(num)

new_list = []

for num in num_list1:
	if num not in new_list:
		new_list.append(num)
```

这里用到了`if`语句中的`in`运算符,作用就是判断`num`在`new_list`中是否已经存在

- 优点:方便好写,可读性强
- 缺点:时间复杂度太差,过于暴力

2. 解法2:个人解法

```python
num_list1 = [19,23,54,64,875,20,109,232,123,54]

num_list2 = [55,80,72,35,60,123,54,29,91]

for i in num_list1:
    num_list2.append(i)
    
for i in range(len(num_list2)):

    for m in range(len(num_list2) - 1,i,-1):
    
        if num_list2[i] == num_list2[m]:
            num_list2.pop(m)

print(f"去重后:{num_list2}")
```

`for m in range(len(num_list2) - 1,i,-1):`是怎么来的?为什么用反向遍历?

因为如果正向遍历,外层/内层循环的 `range `在循环开始时就确定了索引序列，但循环里用了 `pop() `会把列表长度缩短并把后面的元素向左移动。于是原本计划访问的某个索引 m 可能在执行到那一步时已不再存在，导致越界或错误行为。

对于以上两种解法的将两个列表合并的操作,也可以使用Python自带的**解包**操作:

```python
num_list1 = [19,23,54,64,875,20,109,232,123,54]

num_list2 = [55,80,72,35,60,123,54,29,91]

num_list = [*num_list1,*num_list2]
```

解包操作简单理解就是将列表中的数据从列表中拆出来,然后分别操作

其实也可以用`+`直接合并,即:

`num_list = num_list1　+ num_list2`
