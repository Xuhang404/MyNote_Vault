给你两个字符串 `haystack` 和 `needle` ，请你在 `haystack` 字符串中找出 `needle` 字符串的第一个匹配项的下标（下标从 0 开始）。如果 `needle` 不是 `haystack` 的一部分，则返回  `-1` 。

**示例 1：**

**输入：haystack = "sadbutsad", needle = "sad"
输出：**0
解释：**"sad" 在下标 0 和 6 处匹配。
第一个匹配项的下标是 0 ，所以返回 0 。

**示例 2：**

**输入：haystack = "leetcode", needle = "leeto"
输出：**-1
**解释：**"leeto" 没有在 "leetcode" 中出现，所以返回 -1 。

---

在这里我们将`haystack`称为主串,`needle`称为模式串
# 解法一:暴力匹配

将主串的每一个字符与模式串进行匹配,一旦匹配失败,就跳回主串的下一个字符重新匹配

```Java
class Solution {

    public int strStr(String haystack, String needle) {
    
        int nL = needle.length();
        int hL = haystack.length();
        
        for (int i = 0; i <= hL - nL; i++) {
            for (int m = 0; m < nL && haystack.charAt(i + m) == needle.charAt(m); m++) {
                if (m == nL - 1) {
                    return i;
                }
            }
        }
        return -1;
    }
}
```

# 解法二:KMP算法

该算法可以高效地判断字符串是否包含另一个字符串

KMP算法致力于**跳过"==*跳回主串的下一个字符*=="过程**,即我们希望==主串的指针永远向前移动==

我们将遍历主串的指针称为`sCur`,遍历模式串的指针称为`pCur`

假设有下列两个字符串:

- 主串`S`:  `A B C A B C A B D`
- 模串`P`:  `A B C A B D`

当发生不匹配时,`sCur`不回退,我们假设将`P`右移一位,变为:

- 主串`S`:   `A B C A B C A B D`
- 模串`P`:      `A B C A B D`

为了让`A B C A`,即`P[0,3]`能够匹配`sCur`之前的字符串,即`S[1,4]`

那么`P[0,3]`就需要等于`S[1,4]`

我们又知道,`S[1,4] = P[1,4]`

所以只要`P[0,3] = P[1,4]`,模式串右移一位之后,`pCur`之前的字符就能`sCur`之前的字符匹配上了

![[Pasted image 20260610140811.png]]

而`P[0,3]`是子串`P[0,4]`长度为4的**前缀**,`P[1,4]`是子串`P[0,4]`长度为4的后缀

由此我们将问题转变为判断模式串子串的前缀和后缀是否相等 

暂时总结一下:

当发生不匹配时,

此时`pCur`为5,`pCur`之前的子串为`P[0,4]`长度为5

如果`P[0,4]`长度为4的前缀和后缀相等,就能将模式串右移一位,这样`pCur`和`sCur`之前的字符就能匹配成功

>[!注意]
>1. 由于代码层面没有将模式串右移的概念,所以实际上是将`pCur`左移一位
>2. 前缀后缀不包含母串本身,若字符串长度为5,前后缀最长为4

在这个例子中,`P[0,4]`长度为4的前后缀不相等,所以右移一位肯定匹配不上

同理,以此类推

如果`P[0,4]`长度为3的前缀和后缀相等,就能将模式串右移两位,这样`pCur`和`sCur`之前的字符就能匹配成功

在这个例子中,`P[0,4]`长度为3的前后缀不相等,所以右移两位肯定匹配不上

如果`P[0,4]`长度为2的前缀和后缀相等,就能将模式串右移三位,这样`pCur`和`sCur`之前的字符就能匹配成功

在这个例子中,`P[0,4]`长度为2的**前后缀相等,所以右移三位能够匹配成功**

那么==前后缀最大匹配长度==就是2

**如果`P[0.4]`的前后缀最大匹配长度是`n`,就能保证`pCur`后退到`n`,模式串右移`5 - n`位,`pCur`和`sCur`之前的字符能够匹配

也就是说,只要我们知道前后缀最大匹配长度,就可以在发生不匹配时,快速跳到可能匹配到的位置,同时保证跳过的位置肯定不能匹配

![[Pasted image 20260610142131.png]]

通常我们用`next`数组来记录前后缀最大匹配长度

`next[i]`表示子串`P[0,i-1]`的前后缀最大匹配长度,即`next`数组的每一个位置,表示模式串在这个位置的字符之前的子串的前后缀最大匹配长度

![[Pasted image 20260610142430.png]]

重新走一遍流程:

正常遍历到`sCur`和`pCur`等于5,发生第一次不匹配

此时读取`next[5] = 2`,那么`pCur`就回退到2,即模式串右移三位

![[Pasted image 20260610142554.png]]

回退后继续比较,直到匹配成功或字符串结束
