给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出 **和为目标值** _`target`_  的那 **两个** 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。

你可以按任意顺序返回答案。

**示例 1：**

**输入：nums = \[2,7,11,15], target = 9
输出：**\[0,1]
**解释：因为 nums\[0] + nums\[1] == 9 ，返回\[0, 1] 。

**示例 2：**

**输入：nums = \[3,2,4], target = 6
输出：**\[1,2]

**示例 3：**

**输入：nums = \[3,3], target = 6
输出：\[0,1]

## 解法一.暴力枚举

看到这道题的第一个想法就是双重循环,外层循环遍历数组,内层循环寻找对于当前索引`i`,数组内是否存在`target - i`,如果有就返回

```Java
class Solution{

    public int[] twoSum(int[] nums,int target) {    
        for(int i = 0; i < nums.length; i++){
            for(int m = i + 1; m < nums.length; m++){
                if(nums[m] == target - nums[i]){
                    return new int[]{i,m};
                }  
            }
        }
        throw new IllegalArgumentException("No two sum solution");
    }

}
```

但是这种方法双层循环导致其时间复杂度是O(N^2),有没有时间复杂度更低的解法呢?

想得到更短的时间,就要用**空间换时间**,就要找一种更高级的数据结构来存储

现在费时间的原因是嵌套了第二层循环,它的作用是查询`target - nums[i]`,那么我们就用一种查询更快的数据结构就可以了

## 解法二:哈希表

使用哈希表,对于数组的每一个元素`nums[i]`,将它的值`nums[i]`存储为`HashMap`的`Key`,索引`i`存储为`HashMap`的`Value`

这样我们在遍历数组时就可以先判断哈希表中是否存在`target - nums[i]`这个键

如果存在,就用`get()`方法得到这个键对应的数组索引,然后做返回;

如果不存在,就将`<nums[i],i>`这个键值对存入哈希表

```Java
class Solution {  
    public int[] twoSum(int[] nums, int target) {  
       Map<Integer, Integer> map = new HashMap<>();  
       /*把数组的值存为hashmap的key,索引存为hashmap的value  
        *先看hashmap中存不存在target - nums[i]这个键  
        *如果存在,就可以返回get(target - nums[i]),因为通过这个key查询出来的是数组的索引  
        *如果不存在,就把nums[i]存入  
        */       
        for (int i = 0; i < nums.length; i++) {  
          if (map.containsKey(target - nums[i])) {  
             return new int[]{i,map.get(target - nums[i])};  
          } else  
             map.put(nums[i], i);  
       }  
       throw new IllegalArgumentException("No two sum solution");  
    }  
}
```
